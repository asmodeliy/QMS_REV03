1

기간 설정 — 상단에서 날짜와 주차 단위를 선택합니다. (기본값: 현재 주부터 4주)

2

상태 필터 — "{{ t('weekly.all', locale) }}", "{{ t('dashboard.in_progress', locale) }}", "{{ t('task.complete', locale) }}", "{{ t('dashboard.not_started', locale) }}", "{{ t('dashboard.na', locale) }}" 버튼을 사용하여 빠르게 필터링합니다.

  • 행 (세로) — 각 주차의 범위 (예: W01, 2025-01-06 ~ 2025-01-12)

💡 팁: 여러 프로젝트의 일정을 한눈에 비교하고 싶다면 주차 단위를 최대 12주까지 확대할 수 있습니다.

A view that compares weekly schedules based on due dates at a glance.

Weekly Timeline: Weekly Schedule Comparison

Usage

1

Set Period — Select the date and week unit at the top. (Default: 4 weeks from the current week)

2

Status Filter — Quickly filter using the "{{ t('weekly.all', locale) }}", "{{ t('dashboard.in_progress', locale) }}", "{{ t('task.complete', locale) }}", "{{ t('dashboard.not_started', locale) }}", "{{ t('dashboard.na', locale) }}" buttons.

3

{{ t('common.search', locale) }} — Enter project or task names to find them.

Screen Composition

  • Row (Vertical) — Range of each week (e.g., W01, 2025-01-06 ~ 2025-01-12)
  • Column (Horizontal) — Each project (with unique color per project)
  • Cell — Displays cards for tasks due in that week.
💡 Tip: If you want to compare schedules of multiple projects at a glance, you can expand the week unit up to 12 weeks.

{% if locale == 'ko' %}

빠른 입력

  • Tab — 입력 필드 간 이동 (관리자 태스크 테이블)
  • Enter — 행 저장 및 다음 줄로 이동
  • Esc — 편집 취소 및 모달 닫기

효율적인 작업

  • 자동 저장 — STATUS와 REMARK는 입력하면서 자동으로 저장됩니다. 저장 버튼을 클릭할 필요가 없습니다.
  • 프로젝트 포커스 — 관리자에서 작업을 완료한 후 돌아오면 마지막으로 선택한 프로젝트가 유지됩니다.
  • 색상 코딩 — 각 프로젝트는 고유한 색상으로 할당되어 빠른 시각적 스캔이 가능합니다. (대시보드와 주간 보드에서 일관됨)

주간 타임라인 사용

  • 상단의 날짜 입력을 사용하여 특정 주부터의 일정을 빠르게 확인할 수 있습니다.
  • 상태 필터를 조합하여 예를 들어 "이번 주의 진행 중인 항목만 보기"를 할 수 있습니다.
  • 검색 상자에 프로젝트 코드나 태스크 이름을 입력하여 관련 항목만 표시합니다.
  • 각 태스크 우측의 ⋯ 버튼을 클릭하면 상태를 빠르게 변경할 수 있습니다. (Weekly 보드에서 바로 갱신)
{% else %}

Quick Input

  • Tab — Move between input fields (Admin's Task table)
  • Enter — Save row and move to the next line
  • Esc — Cancel editing and close modal

Efficient Work

  • Auto-save — STATUS and REMARK are saved automatically as you type. No need to click a save button.
  • Project Focus — When you return to the admin after working, the last selected project is retained.
  • Color Coding — Each project is assigned a unique color for quick visual scanning. (Consistent across Dashboard and Weekly)

Weekly Timeline Usage

  • Use the Date Input at the top to quickly view schedules starting from a specific week.
  • You can combine Status Filters to, for example, "View only in-progress items for this week".
  • Enter project codes or task names in the Search Box to display only related items.
  • Click the ⋯ button on the right of each task to quickly change its status directly from the Weekly board.

{% if locale == 'ko' %}

Q. 태스크를 어떻게 추가하나요?

관리자 페이지에서 프로젝트를 선택한 후 각 카테고리 블록 오른쪽 상단의 버튼을 사용하여 태스크를 추가합니다.

Q. 마감일은 어디서 설정하나요?

관리자 페이지에서 태스크의 마감일을 설정합니다. 프로젝트 상세 테이블의 DUE 필드는 읽기 전용입니다.

Q. 파일을 첨부했는데 표시되지 않습니다.

STATUS 또는 REMARK를 한 번 업데이트하면 첨부 파일 목록이 새로고침됩니다. 또는 페이지를 다시 로드해보세요.

Q. 색상이 프로젝트마다 고정되나요?

네, 색상은 프로젝트 ID를 기반으로 자동으로 할당되며 변경할 수 없습니다.

Q. 주간 타임라인에서 과거 날짜를 어떻게 보나요?

상단의 날짜 입력에 원하는 주차의 월요일을 입력하고 "조회"를 클릭하면 해당 주부터의 일정을 볼 수 있습니다.

{% else %}

Q. How do I add a task?

Select a project on the Admin Page, then add a task using the button at the top right of each category block.

Q. Where do I set the due date?

Set the due date for tasks on the Admin Page. The DUE field in the Project detail table is read-only.

Q. I attached a file but it doesn't show up.

The attachment list refreshes when you update STATUS or REMARK once. Or try reloading the page.

Q. Are colors fixed per project?

Yes, colors are automatically assigned based on the project ID and cannot be changed.

Q. How do I view past dates in the Weekly Timeline?

Enter the Monday of the desired week in the date input at the top and click "Query" to see the schedule starting from that week.

{% endif %}
