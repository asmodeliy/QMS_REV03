SVIT (시스템 검증 이슈 추적)는 시스템 검증 과정에서 발견되는 이슈를 효과적으로 관리하고 추적하는 도구입니다.

📝 이슈 생성

시스템 검증 중 발견된 문제를 빠르게 기록하고 추적하세요.

🔄 상태 관리

이슈의 진행 상황을 New → In Progress → Under Review → Resolved로 추적하세요.

⚠️ 우선순위 설정

Critical, High, Medium, Low 우선순위로 이슈의 중요도를 구분하세요.

💡 팁: 각 이슈는 고유한 ID (예: MPW27_0001)로 관리되므로 추적과 참조가 용이합니다.
{% else %}

SVIT (System Verification Issue Tracking) is a tool for effectively managing and tracking issues discovered during system verification processes.

📝 Create Issues

Quickly record and track problems discovered during system verification.

🔄 Manage Status

Track issue progress through New → In Progress → Under Review → Resolved.

⚠️ Set Priority

Distinguish issue importance with Critical, High, Medium, Low priority levels.

💡 Tip: Each issue is managed with a unique ID (e.g., MPW27_0001) for easy tracking and reference.
{% endif %}

🔄 {{ t('svit.help_issue_workflow', locale) }}

{% if locale == 'ko' %}

SVIT의 이슈는 다음과 같은 라이프사이클을 따릅니다:

1
이슈 생성

"+ New Issue" 버튼을 클릭하여 새 이슈를 생성합니다. 제목, 현상, 설명, 심각도 등의 필수 정보를 입력합니다.

2
분석 및 진행

이슈를 선택하고 상태를 "In Progress"로 변경하여 문제 분석을 시작합니다.

3
검토 중

분석 완료 후 상태를 "검토 중 (Under Review)"로 변경하여 검토를 요청합니다. 검토자는 첨부된 로그와 메모를 검토하고, 필요시 재현 절차를 요청합니다.

4
해결

이슈가 해결되었으면 상태를 "Resolved"로 변경합니다.

{% else %}

SVIT issues follow the following lifecycle:

1
Create Issue

Click the "+ New Issue" button to create a new issue. Enter required information such as title, phenomenon, description, and severity.

2
Analyze and Progress

Select the issue and change its status to "In Progress" to start analyzing the problem.

3
Wait for Review

After analysis is complete, change the status to "Under Review" to request review. The reviewer will confirm analysis, request additional data if needed, and then approve or ask for more work.

4
Resolve

Once the issue is resolved, change its status to "Resolved".

{% endif %}

각 이슈 상태의 의미를 이해하세요:

상태 설명 다음 단계
{{ t('svit.status_new', locale) }} 새로 생성된 이슈. 아직 분석이 시작되지 않은 상태입니다. 담당자가 할당되어야 합니다. In Progress
{{ t('svit.status_in_progress', locale) }} 현재 분석이 진행 중인 상태입니다. 원인 분석, 재현 테스트, 임시 해결책 적용 등이 진행됩니다. Under Review
{{ t('svit.status_resolved', locale) }} 이슈가 완전히 해결된 상태입니다. 최종 결과와 해결 방법이 기록됩니다. -

상태별 주요 활동

📋 New 상태에서:

  • 이슈 정보의 완전성 확인 (제목, 현상, 재현 단계, 심각도)
  • 담당자(Owner) 지정 및 초기 우선순위 설정
  • 관련 MPW/Node/환경 정보 기입

🔧 In Progress 상태에서:

  • 원인 분석 및 재현 절차 문서화
  • 로그, 스크린샷, 테스트 파일 첨부 (파일 이름 규칙 권장: MPWID_timestamp_type.log)
  • 임시 해결(워크어라운드) 적용 및 결과 기록
  • 분석 메모에 시간/테스트 조건을 기록

✅ 검토 중 (Under Review) 상태에서:

  • 검토자는 분석 내용을 확인하고, 재현 또는 추가 로그를 요청할 수 있습니다
  • 승인 시 상태를 "Resolved"로 변경하거나, 추가 작업 요청 시 "In Progress"로 되돌립니다
{% else %}

Understand the meaning of each issue status:

Status Description Next Step
{{ t('svit.status_new', locale) }} Newly created issue. Analysis has not yet started. An owner should be assigned. In Progress
{{ t('svit.status_in_progress', locale) }} Analysis is currently in progress. Root cause analysis, reproduction testing, and temporary fixes are being implemented. Under Review
{{ t('svit.status_resolved', locale) }} Issue is completely resolved. Final results and resolution methods are recorded. -

Key Activities by Status

NEW

New

Initial triage and owner assignment.

  • Verify issue information completeness (title, phenomenon, env).
  • Assign owner and initial priority.
IN

In Progress

Active analysis and testing.

  • Perform root cause analysis and document steps.
  • Attach logs, screenshots, and test files.
  • Record timestamps and test conditions.
OK

Resolved

Final verification and closure.

  • Confirm fix and update resolution notes.
  • Attach final test results and regression checks.

Reviewer checklist

  • Confirm analysis logs and reproduction steps are sufficient.
  • Review attachments (logs, screenshots, test results).
  • Decide clear resolution criteria (cannot reproduce, verified fixed, etc.).
  • Approve or request further investigation with notes.

{% if locale == 'ko' %}

이슈 생성 시 팁

  • 명확한 제목: 이슈의 핵심을 한눈에 알 수 있도록 작성하세요. (예: "MPW27 메모리 누수 - Reset 후 메모리 증가")
  • 상세한 현상: 발생하는 구체적인 현상을 설명하세요. (어떤 모듈에서, 어떤 조건에서)
  • 재현 단계: 문제를 재현할 수 있는 정확한 단계를 명시하세요.
  • 예상 결과: 정상 동작시 예상되는 결과를 기술하세요.
  • 파일 첨부: 로그 파일, 스크린샷, 테스트 패턴 파일 등을 첨부하세요.
  • 환경 정보: MPW ID, Node, IP/IC 코드, 테스트 환경 등을 명시하세요.

상태 관리 팁

  • 적시 업데이트: 이슈의 진행 상황을 매일 또는 주기적으로 업데이트하세요.
  • 상세 메모: 분석 결과, 원인 분석, 임시 해결책 등을 메모에 기록하세요.
  • 담당자 명확화: 담당 인력과 검토자를 명확히 지정하세요.
  • 첨부 자료 관리: 분석 결과 파일, 패치 파일 등을 첨부하세요.

검증 테스트 팁

  • 다양한 환경 테스트: 여러 노드, IP/IC, 전원 조건에서 테스트하세요.
  • 반복 테스트: 재현성을 확인하기 위해 여러 번 반복 테스트하세요.
  • 회귀 테스트: 해결 후 다른 기능이 영향을 받지 않는지 확인하세요.
  • 극한 조건 테스트: 온도, 주파수, 전압의 극한값에서도 테스트하세요.

팀 협업 팁

  • 명확한 커뮤니케이션: 이슈 상태와 진행 상황을 명확히 전달하세요.
  • 정기 검토: 주간 회의에서 "검토 중 (Under Review)" 상태의 이슈를 검토하세요.
  • 원인 공유: 발견된 원인과 해결 방법을 팀과 공유하세요.
{% else %}

Tips for Creating Issues

  • Clear Title: Write a title that clearly shows the essence. (e.g., "MPW27 Memory Leak - Memory Increases After Reset")
  • Detailed Phenomenon: Describe the specific phenomenon. (Which module, under what conditions)
  • Reproduction Steps: Specify the exact steps to reproduce the problem.
  • Expected Result: Describe the expected result of normal operation.
  • Attach Files: Attach log files, screenshots, test pattern files, etc.
  • Environment Info: Specify MPW ID, Node, IP/IC code, test environment, etc.

Status Management Tips

  • Timely Updates: Update issue progress daily or periodically.
  • Detailed Notes: Record analysis results, root cause analysis, temporary fixes in notes.
  • Clear Assignment: Clearly assign responsible personnel and reviewers.
  • Manage Attachments: Attach analysis result files, patch files, etc.

Verification Test Tips

  • Multi-Environment Testing: Test on various nodes, IPs/ICs, and power conditions.
  • Repetitive Testing: Perform multiple repetition tests to confirm reproducibility.
  • Regression Testing: Verify that other functions are not affected after resolution.
  • Extreme Condition Testing: Test at extreme values of temperature, frequency, and voltage.

Team Collaboration Tips

  • Clear Communication: Clearly communicate issue status and progress.
  • Regular Reviews: Review "Under Review" status issues in weekly meetings.
  • Share Solutions: Share discovered root causes and resolution methods with the team.
{% endif %}

Q: 이슈 ID는 어떻게 생성되나요?

A: 이슈 ID는 셔틀 코드와 생성 순서로 자동 생성됩니다. (예: MPW27_0001) 각 셔틀별로 순번이 독립적으로 증가합니다.

Q: 새로운 이슈를 생성하려면 어떻게 해야 하나요?

A: 메인 페이지의 "+ New Issue" 버튼을 클릭하면 이슈 생성 팝업이 나타납니다. 필수 항목(제목, 현상, 설명)을 입력하고 제출하면 됩니다.

Q: 이슈를 수정할 수 있나요?

A: 이슈를 클릭하여 상세 페이지를 열면 대부분의 필드를 수정할 수 있습니다. 상태, 메모, 파일 첨부 등이 가능합니다.

Q: 여러 파일을 첨부할 수 있나요?

A: 네, 파일 첨부 영역에 여러 파일을 드래그 앤 드롭하거나 파일 선택으로 여러 파일을 첨부할 수 있습니다. 각 파일은 개별적으로 다운로드할 수 있습니다.

Q: 이슈의 상태를 되돌릴 수 있나요?

A: 네, 이슈 상세 페이지에서 상태 드롭다운을 통해 이전 상태로 변경할 수 있습니다. 예를 들어 "검토 중 (Under Review)"에서 "In Progress"로 되돌릴 수 있습니다.

Q: 이슈를 검색하거나 필터링할 수 있나요?

A: 메인 페이지의 필터 영역에서 Shuttle ID, Node, IP/IC, Family로 필터링할 수 있습니다. 필터 조건을 적용하면 일치하는 이슈만 표시됩니다.

Q: 통계 정보는 어떻게 확인하나요?

A: 메인 페이지 상단의 통계 카드에서 전체 이슈 수, NEW, IN_PROGRESS, RESOLVED 상태별 이슈 수를 확인할 수 있습니다.

Q: 이슈 분석 중에 로그 파일을 어떻게 첨부하나요?

A: 이슈 상세 페이지의 "파일 첨부" 섹션에서 로그 파일을 드래그 앤 드롭하거나 파일 선택으로 업로드할 수 있습니다. 최대 파일 크기는 50MB입니다.

Q: 팀원들과 분석 결과를 어떻게 공유하나요?

A: 이슈의 메모 필드에 분석 결과를 작성하고, 필요한 파일을 첨부하세요. 상태를 "검토 중 (Under Review)"로 변경하면 검토자에게 알림이 전달됩니다.

Q: 환경 조건 정보(전력, 온도, 주파수)는 어디에 입력하나요?

A: 이슈 상세 페이지의 "⚡ Condition Parameters" 섹션에서 AVDD, DVDD, VDD, Temperature, Frequency, Pattern 등을 입력할 수 있습니다.

{% else %}

Q: How is the Issue ID generated?

A: The Issue ID is automatically generated from the shuttle code and creation sequence. (e.g., MPW27_0001) The sequence increases independently for each shuttle.

Q: How do I create a new issue?

A: Click the "+ New Issue" button on the main page to open the issue creation popup. Fill in the required fields (title, phenomenon, description) and submit.

Q: Can I edit an issue?

A: Yes, you can edit most fields by clicking on an issue to open its detail page. You can modify status, notes, attachments, etc.

Q: Can I attach multiple files?

A: Yes, you can attach multiple files by dragging and dropping them into the attachment area or by file selection. Each file can be downloaded individually.

Q: Can I revert the issue status?

A: Yes, you can change the status to a previous state using the status dropdown on the issue detail page. For example, you can revert from "Under Review" to "In Progress".

Q: Can I search or filter issues?

A: Yes, you can filter issues by Shuttle ID, Node, IP/IC, and Family in the filter area on the main page. Applying filter conditions will display only matching issues.

Q: How do I view statistics?

A: You can view the total number of issues and the number of issues by status (NEW, IN_PROGRESS, RESOLVED) in the statistics cards at the top of the main page.

Q: How do I attach log files during issue analysis?

A: You can upload log files by dragging and dropping them into the "Attachment" section on the issue detail page or by file selection. Maximum file size is 50MB.

Q: How do I share analysis results with team members?

A: Write the analysis results in the notes field of the issue and attach necessary files. Change the status to "Under Review" to notify the reviewer.

Q: Where do I enter environment condition information (power, temperature, frequency)?

A: You can enter AVDD, DVDD, VDD, Temperature, Frequency, Pattern, etc. in the "⚡ Condition Parameters" section on the issue detail page.
