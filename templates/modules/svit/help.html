📋 SVIT 개요

{% if locale == 'ko' %}

SVIT (System Verification Issue Tracking)는 시스템 검증 및 테스트 과정에서 발견된 이슈를 효과적으로 관리하고 추적하는 통합 도구입니다.

주요 특징

  • 🆔 자동 ID 생성: 각 이슈는 고유한 ID로 관리되어 추적과 참조가 용이합니다.
  • 🔄 상태 관리: New → In Progress → Under Review → Resolved 상태로 진행도를 관리합니다.
  • ⚠️ 우선순위 설정: Critical, High, Medium, Low 4단계로 이슈 중요도를 구분합니다.
  • 👥 담당자 지정: 각 이슈에 담당자를 할당하여 책임을 명확히 합니다.
  • 📎 첨부파일 지원: 로그, 메모리 덤프, 스크린샷 등을 첨부하여 문제 분석을 도와줍니다.
{% else %}

SVIT (System Verification Issue Tracking) is a comprehensive tool for managing and tracking issues discovered during system verification and testing processes.

Key Features

  • 🆔 Auto ID Generation: Each issue gets a unique ID for easy tracking and reference.
  • 🔄 Status Management: Track progress from New → In Progress → Under Review → Resolved.
  • ⚠️ Priority Levels: Classify issues by Critical, High, Medium, or Low importance.
  • 👥 Assignee: Assign owners to each issue to ensure clear responsibility.
  • 📎 Attachments: Support logs, dumps, and screenshots for better analysis.
{% endif %}

🚀 {{ t('svit.help_quick_start', locale) }}

{% if locale == 'ko' %}

1단계: 새 이슈 생성

  • 우측 상단의 "+ New Issue" 버튼을 클릭합니다.
  • 필수 정보를 입력합니다:
    • 제목: 이슈의 간단한 설명 (예: "Login page 버튼이 동작하지 않음")
    • 설명: 상세한 재현 방법, 예상 동작, 실제 동작 등
    • 시스템: 이슈가 발생한 시스템 선택
    • 우선순위: Critical / High / Medium / Low
  • 필요시 로그, 스크린샷 등의 첨부파일을 추가합니다.
  • "Create" 버튼을 클릭하여 이슈를 생성합니다.

2단계: 이슈 할당 및 추적

  • 생성된 이슈의 상세 페이지에서 담당자를 지정합니다.
  • 상태를 업데이트하여 진행 상황을 표시합니다:
    • New: 새로 등록된 이슈
    • In Progress: 분석/수정 중
    • Under Review: 검증/테스트 대기중
    • Resolved: 해결됨
  • 필요시 댓글로 진행 상황을 공유합니다.

3단계: 이슈 조회 및 필터링

  • 우측 상단의 필터 버튼으로 상태, 우선순위, 담당자별로 필터링합니다.
  • 검색창에 이슈 ID나 키워드를 입력하여 빠르게 찾을 수 있습니다.
{% else %}

Step 1: Create New Issue

  • Click the "+ New Issue" button at top right.
  • Fill in required information:
    • Title: Brief description (e.g., "Login button not responsive")
    • Description: How to reproduce, expected vs actual behavior
    • System: Select affected system
    • Priority: Critical / High / Medium / Low
  • Add attachments (logs, screenshots) if needed.
  • Click "Create" to submit.

Step 2: Assign & Track

  • Assign the issue to team members on the detail page.
  • Update status to track progress.
  • Add comments to share updates.

Step 3: Filter & Search

  • Use filters for status, priority, and assignee.
  • Search by issue ID or keywords.
{% endif %}

이슈 생명 주기

각 이슈는 다음과 같은 상태 전환을 따릅니다:

New → In Progress → Under Review → Resolved
  • New (새 이슈)
    • 방금 등록된 이슈입니다.
    • 팀 리더가 내용을 검토 후 담당자를 지정합니다.
  • In Progress (진행 중)
    • 담당자가 이슈 해결을 위해 작업 중입니다.
    • 진행 상황을 주기적으로 댓글로 업데이트합니다.
  • Under Review (검증 대기)
    • 수정/해결이 완료되고 검증을 기다리는 상태입니다.
    • 테스터가 이슈를 재현 및 검증합니다.
  • Resolved (해결됨)
    • 이슈가 완전히 해결되고 검증된 상태입니다.
    • 해결 방법과 최종 결과를 기록합니다.
{% else %}

Issue Lifecycle

Issues follow this state transition:

New → In Progress → Under Review → Resolved
  • New
    • Newly created issue awaiting review.
    • Team leader reviews and assigns an owner.
  • In Progress
    • Assignee is actively working on the issue.
    • Provide regular updates in comments.
  • Under Review
    • Fix/solution is ready for validation.
    • Tester reproduces and verifies the fix.
  • Resolved
    • Issue is completely resolved and verified.
    • Document resolution method and results.
{% endif %}

  • 담당자: 이슈를 해결할 담당자

Basic Information

  • Title: Concise description of the core issue (recommended: under 50 characters)
  • Description: Detailed description, reproduction steps, expected vs. actual behavior
  • System: Select the affected system
  • Priority: Critical / High / Medium / Low

Assignment

  • Assignee: Person responsible for fixing the issue
  • Reporter: Person who found the issue (auto-filled)

Tracking

  • Status: New / In Progress / Under Review / Resolved

  • Last Updated: Last modification date (auto)

✅ {{ t('svit.help_best_practices', locale) }}

{% if locale == 'ko' %}

이슈 등록시 베스트 프랙티스

  • 명확한 제목 작성: "Bug: X page not loading" 대신 "X page fails to load when visiting with admin role"처럼 구체적으로
  • 재현 방법 상세히 기록: 다른 사람도 쉽게 이슈를 재현할 수 있도록
  • 첨부파일 준비: 로그, 스크린샷, 에러 메시지 등을 함께 제공
  • 우선순위 정확히 설정: 서비스 영향도와 심각성을 고려
  • 정기적으로 상태 업데이트: 담당자는 진행 상황을 주 1회 이상 업데이트

우선순위 가이드

  • 🔴 Critical: 전체 시스템 작동 불가, 데이터 손상 위험
  • 🟠 High: 주요 기능 가동 불가, 사용자 작업 차단
  • 🟡 Medium: 특정 기능 미작동, 임시방안 있음
  • 🟢 Low: 부가 기능 미작동, 사용자 영향 미미
{% else %}

Best Practices for Issue Reporting

  • Clear Titles: Be specific and descriptive
  • Detailed Reproduction Steps: Others should easily reproduce the issue
  • Attach Evidence: Include logs, screenshots, error messages
  • Accurate Priority: Consider impact and severity
  • Regular Updates: Assignee should update status at least weekly

Priority Guidelines

  • 🔴 Critical: Complete system failure, data loss risk
  • 🟠 High: Major feature unavailable, user blocked
  • 🟡 Medium: Feature unavailable, workaround exists
  • 🟢 Low: Minor feature affected, minimal user impact
{% endif %}

❓ {{ t('help.faq', locale) }}

{% if locale == 'ko' %}

Q: 이슈에 댓글을 달 수 있나요?

A: 네, 이슈 상세 페이지의 "댓글" 섹션에서 언제든 댓글을 달 수 있습니다. 진행 상황 업데이트나 추가 정보 공유에 유용합니다.

Q: 이슈를 닫을 수 있나요?

A: 이슈는 상태를 "Resolved"로 변경할 수 있습니다. 해결 방법과 검증 결과를 기록한 후 담당자가 설정합니다.

Q: 이슈 ID는 무엇을 의미하나요?

A: 각 이슈는 고유한 ID (예: SVIT-2025-001)로 관리됩니다. 이를 사용하여 이슈를 추적하고 참조할 수 있습니다.

Q: 파일을 첨부할 때 제한이 있나요?

A: 파일명에는 특수문자(#, ?, %)를 사용하지 않기를 권장합니다. 최대 파일 크기는 설정에 따라 다릅니다.

{% else %}

Q: Can I add comments to an issue?

A: Yes, you can add comments in the "Comments" section on the issue detail page. Great for updates and additional information sharing.

Q: Can I close an issue?

A: You can change the status to "Resolved". Document the solution and verification results before marking it resolved.

Q: What does the Issue ID mean?

A: Each issue has a unique ID (e.g., SVIT-2025-001) for tracking and reference purposes.

Q: Are there file upload restrictions?

A: Avoid special characters (#, ?, %) in filenames. Maximum file size depends on system configuration.

{% endif %}
