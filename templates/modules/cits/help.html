📋 {% if locale == 'ko' %}CITS 개요{% else %}CITS Overview{% endif %}

{% if locale == 'ko' %}

CITS (Customer Issue Tracking System)는 고객으로부터 들어오는 문제(이슈)를 한곳에서 추적하고 관리하는 통합 시스템입니다.

주요 특징

  • 📥 이슈 수집: 고객 이슈를 빠르게 등록하고 추적합니다.
  • 🔄 상태 관리: OPEN → PENDING → CLOSE 3가지 상태로 진행도를 관리합니다.
  • 👥 담당자 지정: 각 이슈를 팀원에게 할당하여 책임을 명확히 합니다.
  • 📎 첨부파일: 고객 보내온 로그, 메모리 덤프, 스크린샷 등을 첨부합니다.
  • 💬 커뮤니케이션: 내부 팀과 고객 간 커뮤니케이션을 기록합니다.
{% else %}

CITS (Customer Issue Tracking System) is a comprehensive system for tracking and managing issues from customers in one centralized platform.

Key F{% if locale == 'ko' %}빠른 시작{% else %}Quick Start{% endif %}

{% if locale == 'ko' %}1단계: 새 이슈 등록{% else %}Step 1: Register New Issue{% endif %}>Communication: Record communication between internal teams and customers. {% endif %}

🚀 빠른 시작

1단계: 새 이슈 등록

{% if locale == 'ko' %}
  • 우측 상단의 "+ New Issue" 버튼을 클릭합니다.
  • 고객 정보와 문제를 입력합니다:
    • 고객명: 문제를 보고한 고객
    • 제목: 문제의 간단한 설명
    • 설명: 상세한 문제 상황, 재현 방법, 영향도
    • 제품: 문제가 발생한 제품/칩
    • 노드: 해당 노드
  • 고객이 보낸 첨부파일 (로그, 덤프 등)을 첨부합니다.
  • "Create" 버튼을 클릭합니다.

2단계: 이슈 분석 및 할당

  • 등록된 이슈에서:
    • 내용을 검토하고 담당팀/담당자를 지정합니다.
    • 필요시 고객에서 추가 정보를 요청합니다 (댓글 사용).

3단계: 이슈 진행

  • 상태를 "PENDING"으로 변경합니다.
  • 분석/수정 작업을 진행하며 주기적으로 댓글을 업데이트합니다.

4단계: 이슈 종료

  • 문제가 완전히 해결되면 상태를 "CLOSE"로 변경합니다.
  • 해결 방법과 결과를 최종 주석에 기록합니다.
{% else %}
  • Click the "+ New Issue" button at the top right.
  • Enter customer information and problem details:
    • Customer Name: The customer who reported the issue
    • Title: Brief description of the problem
    • Description: Detailed problem, how to reproduce, impact
    • Product: Product/chip where problem occurred
    • Node: Relevant node information
  • Attach files from customer (logs, dumps, etc.)
  • Click the "Create" button.

Step 2: Analyze and Assign

  • On the registered issue:
    • Review the content and assign to the responsible team/person.
    • Request additional information from customer if needed (use comments).

Step 3: Work on Issue

  • Change status to "PENDING".
  • Perform analysis/fixes and periodically update with comments.

Step 4: Close Issue

  • When problem is fully resolved, change status to "CLOSE".
  • Record the solution and results in the final comment.
{% endif %}

{% if locale == 'ko' %}상태 상세 설명{% else %}Detailed Status Descriptions{% endif %}

🔴 {% if locale == 'ko' %}OPEN (열림) - 새로운 이슈{% else %}OPEN (New Issue){% endif %}

  • {% if locale == 'ko' %}방금 등록된 이슈 상태입니다.{% else %}Newly registered issue.{% endif %}
  • {% if locale == 'ko' %}팀 리더나 담당자가 내용을 검토합니다.{% else %}Team leader or assignee reviews the content.{% endif %}
  • {% if locale == 'ko' %}필요시 고객에게 추가 정보를 요청합니다.{% else %}Request additional information from customer if needed.{% endif %}
  • {% if locale == 'ko' %}담당팀/담당자를 지정해야 합니다.{% else %}Assign to responsible team/person.{% endif %}

🟠 {% if locale == 'ko' %}PENDING (대기중) - 진행 중{% else %}PENDING (In Progress){% endif %}

  • {% if locale == 'ko' %}분석 또는 수정 작업이 진행 중인 상태입니다.{% else %}Analysis or fix work is in progress.{% endif %}
  • {% if locale == 'ko' %}담당자가 주기적으로 진행 상황을 업데이트합니다.{% else %}Assignee regularly updates progress.{% endif %}
  • {% if locale == 'ko' %}고객에게 진행 현황을 공유합니다.{% else %}Share progress status with customer.{% endif %}
  • {% if locale == 'ko' %}필요시 고객과 협력하여 문제를 해결합니다.{% else %}Collaborate with customer if needed.{% endif %}

🟢 {% if locale == 'ko' %}CLOSE (종료) - 해결됨{% else %}CLOSE (Resolved){% endif %}

  • {% if locale == 'ko' %}이슈가 완전히 해결되고 고객이 검증한 상태입니다.{% else %}Issue is fully resolved and customer has verified.{% endif %}
  • {% if locale == 'ko' %}해결 방법, 수정사항, 검증 결과를 상세히 기록합니다.{% else %}Detail resolution method, fixes, and verification results.{% endif %}
  • {% if locale == 'ko' %}고객에게 최종 해결 방법을 안내합니다.{% else %}Communicate final solution to customer.{% endif %}

{% if locale == 'ko' %}상태 전환 규칙{% else %}Status Transition Rules{% endif %}

  • {% if locale == 'ko' %}OPEN → PENDING: 분석 시작 시{% else %}OPEN → PENDING: When analysis starts{% endif %}
  • {% if locale == 'ko' %}PENDING → CLOSE: 고객 검증 완료 후{% else %}PENDING → CLOSE: After customer verification{% endif %}
  • {% if locale == 'ko' %}되돌리간: CLOSE에서 PENDING으로 되돌리칠 수 있습니다 (재현 시){% else %}Reopen: Can change CLOSE back to PENDING (if reproduced){% endif %}

❓ {% if locale == 'ko' %}자주 묻는 질문{% else %}FAQ{% endif %}

Q: {% if locale == 'ko' %}이슈를 고객과 공유할 수 있나요?{% else %}Can I share issues with customers?{% endif %}

A: {% if locale == 'ko' %}네, 이슈를 생성한 후 고객에게 이슈 ID와 링크를 전달하면 고객도 진행 상황을 확인할 수 있습니다. 관리자에게 고객 접근 권한을 요청하세요.{% else %}Yes, after creating an issue, share the issue ID and link with customers so they can track progress. Request customer access permission from admin.{% endif %}

Q: {% if locale == 'ko' %}이슈를 재개(Reopen)할 수 있나요?{% else %}Can I reopen an issue?{% endif %}

A: {% if locale == 'ko' %}네, CLOSE 상태의 이슈에서 "Reopen" 버튼으로 다시 PENDING 상태로 변경할 수 있습니다. 고객이 문제를 재현한 경우 사용합니다.{% else %}Yes, use the "Reopen" button to change a CLOSED issue back to PENDING. Use this when customer reproduces the problem.{% endif %}

Q: {% if locale == 'ko' %}고객으로부터 온 로그가 너무 큽니다. 어떻게 하나요?{% else %}What if logs from customer are too large?{% endif %}

A: {% if locale == 'ko' %}고객에게 필요한 부분만 추출해달라고 요청하거나, zip으로 압축하여 여러 파일로 나누어 업로드하세요.{% else %}Ask customer to extract only necessary parts, or compress with zip and upload in multiple files.{% endif %}

Q: {% if locale == 'ko' %}여러 고객 문제가 같은 원인일 수 있나요?{% else %}Can multiple customer issues have the same root cause?{% endif %}

A: {% if locale == 'ko' %}네, 가능합니다. "Related Issues" 기능으로 연관된 이슈들을 링크할 수 있습니다. 관리자에게 연관 관계를 요청하세요.{% else %}Yes, possible. Use "Related Issues" to link related issues. Request relationship setup from admin.{% endif %}

Q: {% if locale == 'ko' %}해결 방법을 다른 팀과 공유하려면?{% else %}How to share solution with other teams?{% endif %}

A: {% if locale == 'ko' %}이슈의 최종 댓글에 해결 방법을 상세히 기록하고, 필요한 팀원을 @태그로 언급하면 알림을 받게 됩니다.{% else %}Detail the solution in the final comment and @-mention relevant team members to notify them.{% endif %}
